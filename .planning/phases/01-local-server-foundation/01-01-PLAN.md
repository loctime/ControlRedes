**Imports y configuración:**
```javascript
require('dotenv').config();
const express = require('express');
const chokidar = require('chokidar');
const path = require('path');
const fs = require('fs');

const app = express();
app.use(express.json());

const PORT = process.env.PORT || 3333;
const WATCH_DIR = path.resolve(__dirname, '..', 'nuevas-publicaciones');
const ARCHIVE_DIR = path.resolve(__dirname, '..', 'publicaciones-anteriores');
```

**Estado del pipeline (en memoria, mutable):**
```javascript
const state = {
  pipeline: 'idle',      // 'idle' | 'active'
  pendingFiles: [],      // string[] — nombres de archivo, no rutas completas
  activeFile: null,      // string | null — archivo procesándose ahora
  chatId: process.env.CHAT_ID || null,
};
```

**Asegurar que existan los directorios requeridos al arrancar:**
```javascript
[WATCH_DIR, ARCHIVE_DIR].forEach(dir => {
  if (!fs.existsSync(dir)) fs.mkdirSync(dir, { recursive: true });
});
```

**Watcher chokidar — solo para sincronizar state.pendingFiles en tiempo real (no dispara pipeline):**
```javascript
const watcher = chokidar.watch('*.html', {
  cwd: WATCH_DIR,
  persistent: true,
  ignoreInitial: false,
  awaitWriteFinish: { stabilityThreshold: 500, pollInterval: 100 },
});

watcher
  .on('add', filename => {
    if (!state.pendingFiles.includes(filename)) {
      state.pendingFiles.push(filename);
    }
  })
  .on('unlink', filename => {
    state.pendingFiles = state.pendingFiles.filter(f => f !== filename);
  });
```

**Endpoints REST:**

GET /api/status — retorna estado completo del pipeline:
```javascript
app.get('/api/status', (req, res) => {
  res.json({
    pipeline: state.pipeline,
    pendingFiles: [...state.pendingFiles].sort(),
    activeFile: state.activeFile,
  });
});
```

GET /api/files — lista los .html en nuevas-publicaciones/ (lectura directa del FS, no del state):
```javascript
app.get('/api/files', (req, res) => {
  const files = fs.readdirSync(WATCH_DIR)
    .filter(f => f.endsWith('.html'))
    .sort();
  res.json({ files });
});
```

GET /api/files/:filename — sirve el HTML (Phase 3 lo usa para renderizar en tab).
SEGURIDAD (T-01-01 y T-01-02): strip de directorios con path.basename Y verificación de que el filepath resuelto está dentro de WATCH_DIR:
```javascript
app.get('/api/files/:filename', (req, res) => {
  const filename = path.basename(req.params.filename); // strip de directorios — evita path traversal
  const filepath = path.resolve(WATCH_DIR, filename);

  // T-01-02: verificar que el filepath resuelto esté dentro de WATCH_DIR
  if (!filepath.startsWith(WATCH_DIR + path.sep) && filepath !== WATCH_DIR) {
    return res.status(404).json({ error: 'Not found' });
  }

  if (!fs.existsSync(filepath)) return res.status(404).json({ error: 'Not found' });
  res.setHeader('Content-Type', 'text/html');
  res.sendFile(filepath);
});
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem → Express (GET /api/files/:filename) | Nombre de archivo viene del path param HTTP; si no se sanitiza permite path traversal |
| localhost → cualquier proceso local | El servidor escucha en 0.0.0.0 por defecto en Express; cualquier proceso local puede llamar los endpoints |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | GET /api/files/:filename | mitigate | Usar `path.basename(req.params.filename)` para strip de directorios antes de construir el filepath; rechazar cualquier filename que contenga `..` o `/` |
| T-01-02 | Information Disclosure | GET /api/files/:filename | mitigate | Verificar que el filepath resuelto esté dentro de WATCH_DIR antes de servirlo: `filepath.startsWith(WATCH_DIR + path.sep)`. Implementado en el código del endpoint (ver action). |
| T-01-03 | Elevation of Privilege | Servidor escucha en todas las interfaces | accept | Fase 1 es localhost-only, sin exposición de red externa. Mitigación formal en fases de deploy si aplica. |
| T-01-04 | Denial of Service | chokidar watcher en directorios grandes | accept | El directorio nuevas-publicaciones/ contendrá decenas de archivos max. Sin riesgo práctico. |
</threat_model>
