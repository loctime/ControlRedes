<objective>
Crear el esqueleto MV3 instalable con service worker, polling al server via `chrome.alarms`, y persistencia total del estado de trabajo en `chrome.storage.local`.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Crear manifest y estructura base de extension</name>
  <action>
Crear `extension/` con `manifest.json`, `service-worker.js` y modulos compartidos. Declarar permisos minimos: `storage`, `alarms`, `tabs`, `scripting`. Declarar host permissions para `http://localhost:3333/*`, `https://www.instagram.com/*`, `https://www.linkedin.com/*`.
  </action>
</task>

<task type="auto">
  <name>Task 2: Persistencia del estado del pipeline</name>
  <action>
Implementar un modulo `state.js` con schema inicial persistido en `chrome.storage.local`, incluyendo al menos: `server`, `pipeline`, `pendingFiles`, `activeFile`, `lastStatusAt`, `folderPermission`, `loginChecks`.
  </action>
</task>

<task type="auto">
  <name>Task 3: Polling del server con chrome.alarms</name>
  <action>
El service worker crea un alarm al instalarse/iniciarse y, cada tick, consulta `GET /api/status`. Guarda el resultado en storage y tolera fallos sin romperse.
  </action>
</task>

</tasks>

<success_criteria>
- `dist/` contiene una extension cargable
- `chrome://extensions` muestra la extension sin errores
- El alarm existe y actualiza `chrome.storage.local` con datos de `/api/status`
- Terminar y reactivar el service worker no pierde el estado persistido
</success_criteria>
